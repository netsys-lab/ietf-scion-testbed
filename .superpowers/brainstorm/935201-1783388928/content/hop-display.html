B — Latency waterfall

One bar per link, length ∝ RTT-to-next-BR. The shaped hop is unmissable from across the booth — the strongest "the packet found the slow link" read. Utilization/queue demoted to a footer line (or revealed on hover), so the secondary metrics are less visible.

C — Table with inline RTT bars

Table rows keep every metric visible; the RTT column becomes an inline bar so the hot hop still jumps out. Middle ground: slightly busier rows, ms figures move to a footer or hover, but nothing is hidden and it stays one row per link.

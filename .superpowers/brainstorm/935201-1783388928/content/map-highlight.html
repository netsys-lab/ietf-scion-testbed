Map treatment — how should the probed path show on the FabricMap?

Same trace (150 → 154 → 155 → 161). The map already uses color for link health bands (steel/amber/red) and warm-white particles for traffic, so the trace treatment must not impersonate either. All three use brass — the existing "selection/chrome" accent that is never a data encoding.

A — Marching dashes

Brass dashed overlay flowing src → dst along the probed links; endpoint stations ringed brass. Constant motion signals "live probing" and shows direction, cheap CSS animation on an overlay path. Adds permanent motion to an already-animated map (particles).

B — Static brass glow

Probed links get a steady brass glow (drop-shadow) over their health color; no added motion. Calmest option, keeps the particle layer as the only animation — but doesn't convey direction or "probing right now", and glows can smear on low-quality booth screens.

C — Probe comet on a brass base

Faint brass base marks the path; a brass comet travels src → dst on each probe tick — literally "watch the probe packet go collect telemetry". Strongest narrative for attendees, reuses the animateMotion approach the particle layer proved. Slightly more implementation than a dashed overlay.
